🏗️ Caching Architecture & Lifecycle Rules
pseudo
RULE LOCAL_SCHEMA_STRUCTURE:
  TABLE master_data_cache {
    entity_type: ENUM
    canonical_id: UUID
    version: STRING (semver)
    data_json: TEXT
    tenant_id: UUID_OR_NULL ('platform_default' for canonical)
    last_synced_at: DATETIME_UTC
    ttl_seconds: INTEGER
    is_active: BOOLEAN
    checksum: SHA256(data_json)
  }

  TABLE cache_metadata {
    entity_type: ENUM
    server_version: STRING
    last_sync_timestamp: DATETIME_UTC
    sync_mode: ENUM[full, delta, event_driven]
    sync_status: ENUM[ok, stale, corrupted, pending]
  }

RULE CACHE_LIFECYCLE:
  ON app_first_launch:
    sync_engine.request_master_data(entity_type: ALL, mode: full)
    cache_metadata.sync_status = "ok"
    
  ON app_startup (subsequent):
    IF cache_metadata.last_sync_timestamp + ttl < current_time THEN
      cache_metadata.sync_status = "stale"
      trigger_background_delta_sync()
    ELSE
      cache_metadata.sync_status = "ok"
      
  ON delta_sync_response:
    FOR EACH item IN response.deltas:
      UPSERT INTO master_data_cache (version, data_json, last_synced_at)
    FOR EACH item IN response.tombstones:
      SET is_active = FALSE WHERE canonical_id = item.id
      
    UPDATE cache_metadata SET server_version = response.version
    cache_metadata.sync_status = "ok"
🔄 Delta Sync & Versioning Protocol
RULE API_CONTRACT:
  GET /api/v1/master-data/sync
    QUERY_PARAMS:
      entity_type: enum
      last_version: string
      tenant_id: string
      
  RESPONSE_STRUCTURE:
    {
      version: "v1.2.0",
      full_sync_required: false,
      deltas: [
        { id: UUID, version: "v1.2.0", payload: JSON, action: "create" | "update" },
        { id: UUID, version: "v1.2.0", action: "deprecate" }
      ],
      tombstones: [ { id: UUID, deprecated_at: ISO8601 } ],
      sync_metadata: { timestamp: UTC, tenant_overrides_count: int }
    }

RULE DELTA_APPLICATION_LOGIC:
  FUNCTION apply_master_delta(deltas[], tombstones[]):
    BEGIN TRANSACTION
    FOR delta IN deltas:
      IF delta.action == "create" OR delta.action == "update" THEN
        INSERT_OR_REPLACE master_data_cache 
          (canonical_id, version, data_json, last_synced_at)
        WHERE canonical_id = delta.id
      ELSE IF delta.action == "deprecate" THEN
        UPDATE master_data_cache SET is_active = FALSE 
        WHERE canonical_id = delta.id
    FOR tombstone IN tombstones:
      UPDATE master_data_cache SET is_active = FALSE, is_deleted = TRUE
      WHERE canonical_id = tombstone.id
    COMMIT
    UPDATE cache_metadata SET server_version = response.version
⏱️ Cache Invalidation & Refresh Rules
pseudo
RULE INVALIDATION_TRIGGERS:
  1. TTL_EXPIRY: automatic background refresh when ttl elapsed
  2. VERSION_MISMATCH: client_version < server_version detected during sync handshake
  3. EVENT_DRIVEN: server publishes `master_data_updated.v1` → mobile wakes sync worker
  4. MANUAL_OVERRIDE: dealer/admin taps "Refresh Master Data" → forces full sync for entity_type
  5. TENANT_CONFIG_CHANGE: workflow/KPI/template update → invalidates related cache partitions

RULE FRESHNESS_VALIDATION:
  FUNCTION is_cache_fresh(entity_type):
    metadata = SELECT * FROM cache_metadata WHERE entity_type = entity_type
    IF metadata IS NULL RETURN false
    IF metadata.sync_status == "corrupted" RETURN false
    IF current_time > metadata.last_sync_timestamp + ttl_seconds RETURN false
    RETURN true

RULE CONCURRENT_UPDATE_HANDLING:
  IF mobile_sync_version < server_version THEN
    mobile downloads delta, applies locally, updates version
    NEVER reject server updates due to local edits (master data is read-only on mobile)
📉 Fallback & Degradation Behavior
pseudo
RULE FALLBACK_HIERARCHY:
  1. Try L2 SQLite cache (is_active = TRUE, version matches or latest available)
  2. If cache empty/stale AND network available → fetch from server, cache, return
  3. If cache empty/stale AND network unavailable → 
     a. Show "⚠️ Using older reference data" banner
     b. Allow workflow to continue with stale cache
     c. Log telemetry: master_data_fallback_triggered
  4. NEVER throw blocking error for missing master data in field workflows

RULE VALIDATION_FALLBACK:
  IF dropdown_data_missing THEN
    USE regex/schema_validation_from_local_rules_registry
    SHOW "Manual entry allowed - will validate on sync"
    QUEUE for async validation once connectivity restored
📱 Mobile Storage & Eviction Rules

RULE STORAGE_BUDGET:
  MAX master_data_cache_size = 50MB per device
  IF total_size > budget THEN execute LRU_eviction_policy

RULE LRU_EVICTION_POLICY:
  ORDER BY last_accessed_timestamp ASC
  EVICT entity_type IN [campaigns, regional_pricing, optional_forms] FIRST
  NEVER EVICT [geography_villages, crop_master, soil_types, canonical_enums]
  PRESERVE tenant_overrides_until explicit_sync_or_logout

RULE COMPRESSION_OPTIMIZATION:
  STORE JSON payload minified
  IF payload > 10KB THEN compress_with_gzip_before_SQLite_insert
  DECOMPRESS_on_read_into memory cache
  TRACK compression_ratio_in telemetry
🔐 Tenant Isolation & Query Logic
pseudo
RULE QUERY_COALESCE_PATTERN:
  FUNCTION get_master_data(entity_type, filter_criteria, tenant_id):
    QUERY:
      SELECT * FROM master_data_cache
      WHERE entity_type = entity_type
        AND is_active = TRUE
        AND tenant_id IN (tenant_id, 'platform_default')
        AND matches(filter_criteria)
      ORDER BY 
        CASE WHEN tenant_id = tenant_id THEN 1 ELSE 0 END DESC,
        version DESC
      LIMIT 1 PER canonical_id

  RESULT: tenant_specific_override_wins_if_exists, else platform_default

RULE ISOLATION_GUARANTEES:
  tenant_overrides MUST include tenant_id in primary key composite
  CROSS_TENANT_QUERY MUST be rejected at API layer
  AUDIT_LOG records all tenant_override_create/update/delete_with_actor_id
